backend/Dockerfile

Optimized Build 
# Stage 1: Build dependencies
FROM python:3.11-slim AS builder
WORKDIR /app
RUN apt-get update && apt-get install -y --no-install-recommends \
    gcc libpq-dev && rm -rf /var/lib/apt/lists/*
COPY requirements.txt .
RUN pip wheel --no-cache-dir --no-deps --wheel-dir /app/wheels -r requirements.txt

# Stage 2: Final production image
FROM python:3.11-slim
WORKDIR /app
RUN apt-get update && apt-get install -y --no-install-recommends \
    libpq5 && rm -rf /var/lib/apt/lists/*
COPY --from=builder /app/wheels /wheels
RUN pip install --no-cache /wheels/*
COPY . /app
RUN useradd -m appuser && chown -R appuser /app
USER appuser
EXPOSE 8000
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

docker-compose.yml

Orchestration 
version: '3.8'
services:
  db:
    build: ./database
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: password
      POSTGRES_DB: webapp
    volumes: [pgdata:/var/lib/postgresql/data]
    networks: [macvlan_net]
  backend:
    build: ./backend
    environment:
      DATABASE_URL: postgresql://postgres:password@db:5432/webapp
    ports: ["8000:8000"]
    networks: [macvlan_net]
    depends_on: { db: { condition: service_healthy } }
networks:
  macvlan_net: { external: true, name: host_macvlan }

02 / Macvlan vs. Ipvlan Comparison

Macvlan Driver

Unique IP/MAC per container. Perception as hardware via 802.1q. Best for on-premise hardware transparency.

Ipvlan Driver

Shared Parent MAC with unique IPs. Critical for AWS/Cloud restrictions where MAC spoofing is blocked.
